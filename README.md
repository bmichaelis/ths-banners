# Timpanogos Timberwolves Banner Pipeline

Automated PDF banner generation for Timpanogos Timberwolves (THS) sponsor banners.

Sponsors email a PNG or JPG logo to a dedicated address. The system picks it up, composites it into the banner template, and emails a download link to the printer (and a CC recipient) — no manual steps required.

## Architecture

```
Sponsor email (PNG/JPG attached)
  |
  v
Cloudflare Email Routing
  --> twolf-banners-worker (email handler)
        --> R2: pending/<uuid>-<filename>
  |
  v (every 15 min, or on-demand)
GitHub Actions: Banner Pipeline
  --> pipeline/process_pending.py
        1. Lists R2 pending/
        2. Downloads logo
        3. Runs make_banner.py  --> <sponsor>-banner.pdf
        4. Uploads to R2: done/<sponsor>-banner.pdf
        5. Generates presigned download URL (7-day expiry)
        6. POST /send-banner --> twolf-banners-worker (HTTP handler)
              --> Resend API --> email to printer + CC
        7. Deletes pending file from R2
```

## Repository

GitHub: `bmichaelis/ths-banners`

## Python Dependencies

- **PyMuPDF** — PDF manipulation
- **Pillow** — image size inspection
- **boto3** — R2 access via S3-compatible API
- **requests** — HTTP calls to the Cloudflare Worker

Install:

```bash
pip install -r requirements.txt
```

## Common Operations

### Generate a banner manually

```bash
python3 make_banner.py logo.png output.pdf
```

The logo is scaled to fit the white space in the banner (centered, with padding) and written into `output.pdf`.

### Recreate the template

Only needed if `NFlooring.pdf` changes (e.g., the base banner layout is updated).

```bash
python3 create_template.py
```

This redacts the existing sponsor logo area from `NFlooring.pdf` and saves `template.pdf`. All subsequent banners are built from `template.pdf`.

### Send a banner via the automated pipeline

1. Email a PNG or JPG logo attachment to `twolfbanners@kindacoach.com`.
2. The Cloudflare Worker saves it to R2 (`pending/`).
3. GitHub Actions picks it up within 15 minutes, generates the banner, and emails the printer.

To process immediately without waiting: go to the GitHub Actions tab and manually trigger the **Banner Pipeline** workflow.

### Run the tests

Python tests:

```bash
python3 -m pytest tests/ -v
```

Cloudflare Worker tests:

```bash
cd cloudflare
npm test
```

### Deploy Worker changes

```bash
cd cloudflare
wrangler deploy
```

### Add or rotate the API token

The API token authenticates GitHub Actions calls to the Worker's `/send-banner` endpoint. Both sides must hold the same value.

1. Generate a new token (any random string works, e.g. `openssl rand -hex 32`).
2. Update the Worker secret:
   ```bash
   cd cloudflare
   wrangler secret put API_TOKEN
   ```
3. Update the GitHub Actions secret `CF_WORKER_API_TOKEN` in the repository settings.

### Update the printer email

Update the `PRINTER_EMAIL` secret in GitHub Actions repository settings.

### Update the CC email

Update the `CC_EMAIL` secret in GitHub Actions repository settings.

## Secrets Reference

### GitHub Actions secrets

| Secret | Purpose |
|---|---|
| `R2_ACCOUNT_ID` | Cloudflare account ID |
| `R2_ACCESS_KEY_ID` | R2 API token key |
| `R2_SECRET_ACCESS_KEY` | R2 API token secret |
| `R2_BUCKET_NAME` | `twolf-banners` |
| `CF_WORKER_SEND_URL` | Worker URL (for `/send-banner`) |
| `CF_WORKER_API_TOKEN` | Bearer token for Worker auth |
| `PRINTER_EMAIL` | Printer's email address |
| `CC_EMAIL` | CC recipient email |

### Cloudflare Worker secrets (set via `wrangler secret put`)

| Secret | Purpose |
|---|---|
| `FROM_EMAIL` | Outbound sender address (`twolfbanners@kindacoach.com`) |
| `RESEND_API_KEY` | Resend API key for sending email |
| `API_TOKEN` | Bearer token matching `CF_WORKER_API_TOKEN` |

## Services

| Service | Purpose |
|---|---|
| GitHub / GitHub Actions | Code hosting; scheduled pipeline (every 15 min) + manual trigger |
| Cloudflare Email Routing | Routes `twolfbanners@kindacoach.com` to the Worker |
| Cloudflare R2 (`twolf-banners`) | Object storage: `pending/` for incoming logos, `done/` for finished banners |
| Cloudflare Workers (`twolf-banners-worker`) | Email handler (inbound) + HTTP `/send-banner` handler (outbound) |
| Resend | Transactional email delivery |

## File Layout

```
NFlooring.pdf           # Source banner (original, with National Flooring Brokers logo)
template.pdf            # Clean banner with white space (generated by create_template.py)
create_template.py      # Strips the sponsor logo area from NFlooring.pdf
make_banner.py          # Inserts a new logo into template.pdf
requirements.txt        # Python dependencies
pipeline/
  process_pending.py    # GitHub Actions entrypoint: polls R2, runs pipeline
tests/                  # Python unit tests
cloudflare/
  wrangler.toml         # Worker config (name: twolf-banners-worker, R2 binding)
  src/
    index.ts            # Worker entry point (email + fetch handlers)
    email-inbound.ts    # Saves PNG/JPG attachments to R2 pending/
    email-outbound.ts   # POST /send-banner: sends email via Resend
    types.ts            # Env interface
  test/                 # Worker unit tests (Vitest)
.github/workflows/
  banner-pipeline.yml   # Scheduled workflow (cron every 15 min) + workflow_dispatch
```
